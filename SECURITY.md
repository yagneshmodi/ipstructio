# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not open a public issue**.  
Instead, report it privately to the maintainers so it can be addressed responsibly.

**Email:** yagneshmodi@proton.me

Please include the following information in your report:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Any relevant screenshots, logs, or error messages
- The environment or configuration where the issue occurs

---

## Supported Versions

| Version | Supported          |
|---------|------------------|
| 1.0.x   | ✅ Yes            |
| 0.x     | ❌ No             |

> Only the latest stable release receives security updates.

---

## Security Updates

- Security fixes will be released promptly as patches or new versions.
- Users are encouraged to **update to the latest version** to mitigate risks.
- All security fixes will be documented in the [CHANGELOG.md](./CHANGELOG.md).

---

## Reporting Guidelines

When submitting a report:

1. Do **not** publicly disclose the vulnerability until it has been addressed.  
2. Provide as much detail as possible for reproducibility.  
3. Maintain confidentiality of any sensitive information involved.

---

## Acknowledgements

We appreciate and recognize security researchers who report vulnerabilities responsibly.  
Your contributions help make this project safer for everyone.
